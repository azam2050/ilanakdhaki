Build a complete Smart Ads Platform with two dashboards:

=== CORE CONCEPT ===
An AI-powered marketing manager for Salla/Zid merchants.
Merchants install the app, connect their ad accounts,
and the AI manages everything automatically —
like a professional marketing agency with no human intervention.

=== HOW IT WORKS ===
1. Merchant installs from Salla app store
2. Grants permission to read store data + share encrypted purchase patterns
3. Connects their own ad accounts (Meta/Snap/TikTok/Google) via OAuth
4. AI analyzes their store + past campaigns from their ad accounts
5. AI launches optimized campaigns FROM their accounts
6. Platform enriches their ad accounts with network audience data
7. AI optimizes every 15 minutes automatically

=== AI ENGINE ===

Optimization Logic (runs every 15 minutes):
Step 1: Math check (no Claude call)
- Is CPO > 1.5× network average for this segment?
- Is CTR < 50% of network average?
- Is budget 80%+ spent?
→ If no issues: do nothing (saves Claude calls)
→ If issue detected: call Claude

Step 2: Claude decides (only when needed):
Send Claude: campaign metrics + network averages for segment
Claude returns JSON:
{
  action: "SCALE_UP|SCALE_DOWN|CHANGE_AUDIENCE|REFRESH_CREATIVE|PAUSE_PLATFORM|KEEP",
  reason_arabic: "سناب أكثر كفاءة صباحاً",
  result_arabic: "النتيجة: ٧ طلبات إضافية",
  params: {}
}

Step 3: Execute immediately via platform API
Step 4: Save to ai_decisions table
Step 5: Show in merchant dashboard (Arabic only)

Daily Tasks (Cron Jobs):
- Every 15 min: optimization check
- Every 24 hours: refresh network audiences for all segments
- Every day at 7am: generate 10 Arabic ad tips per merchant
- Every day: send report (based on merchant preference)
- Every 6 hours: database backup to S3
- Weekly: rebuild Lookalike audiences

Network Audience Logic:
When merchant joins:
1. Pull 12 months of orders from Salla API → hash emails/phones SHA-256
2. Pull network buyers for their segment from ClickHouse
3. Upload Custom Audience to their Meta/Snap accounts
4. Build Lookalike (2% similarity)
5. Send CAPI events to their pixel

When merchant cancels:
1. Stop all CAPI event sending immediately
2. Delete network Custom Audience from their accounts (within 24h)
3. Keep only their own customer audience (it's theirs)
4. Send win-back messages at day 7, 14, 30 showing performance drop

=== SECURITY ===
- All customer emails/phones: SHA-256 before storage (never store plain text)
- All OAuth tokens: AES-256 encrypted (key in environment variable)
- HTTPS only, reject HTTP
- Rate limiting: 100 requests/minute per merchant
- JWT tokens: 1 hour expiry with secure refresh tokens
- Webhook signature verification (HMAC-SHA256) before processing
- Idempotency: process each webhook exactly once
- Database: no external connections, internal only
- Automatic backup every 6 hours to S3 (AES-256 encrypted)

=== TECH STACK ===
- Frontend: Next.js 14 App Router, RTL Arabic, Tailwind CSS
- Backend: NestJS + TypeScript
- Database: PostgreSQL + Prisma ORM
- Cache & Queues: Redis + BullMQ
- Analytics DB: ClickHouse
- AI: Claude API (claude-sonnet-4-5) — call only when needed
- Image Generation: Fal AI
- Hosting: Replit → custom domain (hide all Replit traces)
- Notifications: Firebase Cloud Messaging
- Storage: AWS S3
- Payment: Moyasar (mada, Apple Pay, STC Pay)
